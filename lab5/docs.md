  1. Постановка задачи                                                                                                                                                                                                  
                                                                                                                                                                                                                        
  Path Tracing (трассировка путей) — метод рендеринга, основанный на моделировании переноса света методом Монте-Карло. Для каждого пикселя выпускается набор лучей (samples per pixel, SPP), каждый из которых          
  отслеживает случайный путь света от камеры через bouncing на поверхностях сцены.                                                                                                                                      
                                                                                                                                                                                                                        
  Проблема: При недостаточном числе сэмплов (64 spp) возникает высокочастотный шум. Это особенно заметно в косвенной составляющей освещения (indirect light) — части света, пришедшей после 2+ отражений. Прямое        
  освещение (direct light), вычисленное через NEE (Next Event Estimation — явная выборка источника), шумит меньше.                                                                                                      
                                                                                                                                                                                                                        
  Решение: Применить Joint Bilateral Filter только к indirect_light, используя геометрическую информацию (G-буферы) для предотвращения размытия границ.

  3. Path Tracer: что происходит                                                                                                                                                                                        
                                                                                                                                                                                                                        
  3.1. Пересечение луча с треугольником (алгоритм Мёллера-Трумбора)                                                                                                                                                     
                                                                                                                                                                                                                        
  Для каждого луча нужно найти ближайший треугольник, который он пересекает. Алгоритм выражает точку на луче O + t*D как барицентрическую комбинацию треугольника V0 + u*E1 + v*E2 и решает систему из 3 уравнений:     
                                                                                                                                                                                                                        
  O + t*D = V0 + u*E1 + v*E2                                                                                                                                                                                            
                                                                                                                                                                                                                        
  Решение через определитель Крамера даёт t, u, v. Пересечение валидно если t > 0, u >= 0, v >= 0, u + v <= 1.                                                                                                          
                                                                                                                                                                                                                        
  Для ускорения все данные треугольников хранятся в flat numpy-массивах (_v0f, _e1f, _e2f) — векторе длинной 3*N, где координаты (x,y,z) каждого треугольника лежат по индексам [i, i+1, i+2] (шаг ::3). Это позволяет  
  проверять пересечение со всеми треугольниками одновременно через broadcasting — вместо цикла по 36 треугольникам одна векторизованная операция.                                                                       
                                                                                                                                                                                                                        
  3.2. Трассировка одного пути (trace)                                                                                                                                                                                  
                                                                                                                                                                                                                        
  throughput = [1, 1, 1]   // накопленная вероятность×цвет пути                                                                                                                                                         
  color = [0, 0, 0]        // итоговый цвет пикселя                                                                                                                                                                     
                                                                                                                                                                                                                        
  Для каждого bounce (макс. 8):                                                                                                                                                                                         
    1. Найти пересечение луча со сценой                                                                                                                                                                                 
    2. Если попали на эмиссивную поверхность → добавить emission × throughput                                                                                                                                           
    3. Русская рулетка: после 3 bounce — с вероятностью p_continue                                                                                                                                                      
       путь продолжается, throughput делится на p_continue.                                                                                                                                                             
       Это позволяет ограничить длину пути без внесения смещения.                                                                                                                                                       
    4. Выбрать тип отражения:                                                                                                                                                                                           
       - Диффузное (p_diff):                                                                                                                                                                                            
         а) NEE: явно выбрать источник → direct += nee_value                                                                                                                                                            
         б) Случайное направление (cosine-weighted hemisphere)                                                                                                                                                          
         в) throughput *= diffuse_color                                                                                                                                                                                 
       - Зеркальное (1 - p_diff):                                                                                                                                                                                       
         а) Отражение по закону Снеллиуса                                                                                                                                                                               
         б) throughput *= specular_color                                                                                                                                                                                
                                                                                                                                                                                                                        
  Ключевое разделение для Lab 5:                                                                                                                                                                                        
  - direct — вклад от NEE (явная выборка источника на каждом bounce)                                                                                                                                                    
  - indirect = total - direct — эмиссия при случайном попадании на источник + всё остальное                                                                                                                             
  - NEE компенсация: если после NEE следующий bounce случайно попадает на тот же источник, NEE-вклад вычитается из indirect, чтобы избежать двойного счёта                                                              
                                                                                                                                                                                                                        
  3.3. Next Event Estimation (_direct_light)                                                                                                                                                                            
                                                                                                                                                                                                                        
  На каждом bounce при диффузном отражении:                                                                                                                                                                             
  1. Выбрать случайный треугольник источника с вероятностью, пропорциональной emission × area                                                                                                                           
  2. Выбрать случайную точку на нём                                                                                                                                                                                     
  3. Проверить видимость (shadow ray) — нет ли препятствий                                                                                                                                                              
  4. Вычислить вклад по рендер-уравнению:                                                                                                                                                                               
                                                                                                                                                                                                                        
  L_direct = L_e(light) × (diffuse / π) × (cos_θ_surf × cos_θ_light / r²) / pdf                                                                                                                                         
                                                                                                                                                                                                                        
  Это даёт более чистую оценку прямого освещения по сравнению со случайным ожиданием попадания на источник.

  4. G-буферы: что хранится                                                                                                                                                                                             
                                                                                                                                                                                                                        
  При рендеринге параллельно с цветом накапливаются геометрические данные из первичного луча (первое пересечение):                                                                                                      
  ┌───────────────────┬──────────────────────────────────────────────────────┬───────────────────────────────────────────────────────────────────────────────────────────┐                                              
  │      G-буфер      │                      Что хранит                      │                                 Как используется фильтром                                 │                                              
  ├───────────────────┼──────────────────────────────────────────────────────┼───────────────────────────────────────────────────────────────────────────────────────────┤                                              
  │ depth_map[H,W]    │ Расстояние t от камеры до первой точки попадания     │ G_r_depth: гауссиан от разницы глубин. Не даёт смешивать пиксели с сильно разной глубиной │                                              
  ├───────────────────┼──────────────────────────────────────────────────────┼───────────────────────────────────────────────────────────────────────────────────────────┤                                              
  │ normal_map[H,W,3] │ Нормаль поверхности в точке попадания                │ G_r_normal: гауссиан от 1 - dot(n_p, n_q). Не даёт смешивать пиксели на разных гранях     │                                              
  ├───────────────────┼──────────────────────────────────────────────────────┼───────────────────────────────────────────────────────────────────────────────────────────┤                                              
  │ object_index[H,W] │ Идентификатор поверхности (группировка по материалу) │ G_r_obj: жёсткая маска 0/1. Не даёт смешивать пиксели разных объектов вообще              │                                              
  └───────────────────┴──────────────────────────────────────────────────────┴───────────────────────────────────────────────────────────────────────────────────────────┘                                              
  Почему группировка по материалу, а не по треугольнику: Пол состоит из 2 треугольников. Если дать каждому уникальный индекс, фильтр создаст видимый шов по диагонали. Треугольники с одним материалом = одна           
  поверхность = один индекс. Depth/normal веса всё равно не дадут смешать пол с потолком (у них разные нормали и глубина).

  5. Joint Bilateral Filter: математика                                                                                                                                                                                 
                                                                                                                                                                                                                        
  5.1. Обычный Bilateral Filter                                                                                                                                                                                         
                                                                                                                                                                                                                        
  Классический Bilateral Filter (Tomasi & Manduchi, 1998):                                                                                                                                                              
                                                                                                                                                                                                                        
  g(p) = (1/W_p) × Σ_{q ∈ Ω} f(q) × G_s(||p-q||) × G_r(||f(p)-f(q)||)                                                                                                                                                   
                                                                                                                                                                                                                        
  Проблема: диапазонный вес G_r зависит от разницы цветов — а шум и есть изменение цвета. Фильтр «не понимает», где шум, а где граница объекта.                                                                         
                                                                                                                                                                                                                        
  5.2. Joint/Cross Bilateral Filter — наше решение                                                                                                                                                                      
                                                                                                                                                                                                                        
  Используем не цвет, а G-буферы (которые не содержат шума) в качестве диапазонного веса:                                                                                                                               
                                                                                                                                                                                                                        
  g(p) = (1/W_p) × Σ_{q ∈ S} f(q) × G_s(||p - q||) × G_r(p, q)                                                                                                                                                          
                                                                                                                                                                                                                        
  5.3. Пространственный вес G_s                                                                                                                                                                                         
                                                                                                                                                                                                                        
  G_s(||p - q||) = exp(-(dx² + dy²) / (2 × σ_s²))                                                                                                                                                                       
                                                                                                                                                                                                                        
  Обычный 2D гауссиан. Близкие пиксели получают больший вес. σ_s = 2.5 пикселя, radius = 6 (ядро 13×13).                                                                                                                
                                                                                                                                                                                                                        
  Предвычисляется один раз как матрица (2r+1) × (2r+1).                                                                                                                                                                 
                                                                                                                                                                                                                        
  5.4. Диапазонный вес G_r — произведение трёх компонент                                                                                                                                                                
                                                                                                                                                                                                                        
  G_r(p, q) = G_r_obj × G_r_depth × G_r_normal                                                                                                                                                                          
                                                                                                                                                                                                                        
  a) Индекс объекта — жёсткая граница:                                                                                                                                                                                  
  G_r_obj = 0,  если object_index[p] ≠ object_index[q]                                                                                                                                                                  
           = 1,  иначе                                                                                                                                                                                                  
                                                                                                                                                                                                                        
  Два пикселя на разных объектах никогда не смешиваются. Это двоичная маска — самые жёсткие из возможных весов.                                                                                                         
                                                                                                                                                                                                                        
  b) Разница глубин — мягкая граница:                                                                                                                                                                                   
  G_r_depth = exp(-(z_p - z_q)² / (2 × σ_z²))                                                                                                                                                                           
                                                                                                                                                                                                                        
  Гауссиан от разницы глубин. Если два пикселя на одном объекте, но один на переднем плане, а другой на заднем (например, тень от объекта), вес падает. σ_z = 0.05 — при разнице глубин 0.05 ед. вес падает до e^{-0.5} 
  ≈ 0.6.                                                                                                                                                                                                                
                                                                                                                                                                                                                        
  c) Разница нормалей — мягкая граница:                                                                                                                                                                                 
  G_r_normal = exp(-(1 - dot(n_p, n_q)) / (2 × σ_n²))                                                                                                                                                                   
                                                                                                                                                                                                                        
  Используем 1 - dot(n_p, n_q) как меру угла между нормалями:                                                                                                                                                           
  - dot = 1 → одинаковые нормали → мера = 0 → вес = 1 (максимальный)                                                                                                                                                    
  - dot = 0 → ортогональные нормали → мера = 1 → вес ≈ 0 (почти нулевой)                                                                                                                                                
  - σ_n = 0.08 — уже при угле ~16° (dot ≈ 0.96) вес падает вдвое                                                                                                                                                        
                                                                                                                                                                                                                        
  Это критически важно для сохранения рёбер боксов: соседние грани имеют одинаковый object_index, но нормали развёрнуты на 90°, и G_r_normal не даёт их смешать.                                                        
                                                                                                                                                                                                                        
  5.5. Нормировка — сохранение энергии                                                                                                                                                                                  
                                                                                                                                                                                                                        
  W_p = Σ_{q ∈ S} G_s(||p - q||) × G_r(p, q)                                                                                                                                                                            
                                                                                                                                                                                                                        
  g(p) = (1/W_p) × Σ f(q) × w(p, q)                                                                                                                                                                                     
                                                                                                                                                                                                                        
  Поскольку:                                                                                                                                                                                                            
  - все веса w ≥ 0 (гауссианы и бинарная маска)                                                                                                                                                                         
  - сумма весов = W_p                                                                                                                                                                                                   
  - результат = взвешенное среднее с суммой весов = 1                                                                                                                                                                   
                                                                                                                                                                                                                        
  → Яркость не добавляется и не удаляется, а только перераспределяется между соседними пикселями. Это и есть закон сохранения энергии.                                                                                  
                                                                                                                                                                                                                        
  На практике: Σ_p g(p) ≈ Σ_p f(p) с отклонением < 0.2% (из-за edge padding).

  6. Реализация фильтра — как устроен код                                                                                                                                                                               
                                                                                                                                                                                                                        
  Вместо наивного вложенного цикла for py: for px: for dy: for dx: (медленно), используется цикл по смещениям ядра for dy: for dx:, а внутри — векторизованная операция над всем изображением:                          
                                                                                                                                                                                                                        
  for dy in range(kernel_size):     # 13 итераций                                                                                                                                                                       
      for dx in range(kernel_size): # 13 итераций                                                                                                                                                                       
          # Для ВСЕХ пикселей изображения одновременно:                                                                                                                                                                 
          f_q = noisy_pad[dy:dy+H, dx:dx+W]      # соседи                                                                                                                                                               
          w = gs[dy,dx] * gr_obj * gr_depth * gr_normal  # веса                                                                                                                                                         
          result += f_q * w[:,:,np.newaxis]       # накопление                                                                                                                                                          
          weight_sum += w                          # нормировка                                                                                                                                                         
                                                                                                                                                                                                                        
  Итого: 13 × 13 = 169 итераций вместо 512 × 512 × 169 ≈ 44M. Каждая итерация — одна numpy-операция над массивом 512×512.                                                                                               
                                                                                                                                                                                                                        
  Padded-массивы (дополнение по краям через np.pad(..., mode='edge')) позволяют обращаться к соседям за пределами изображения без проверки границ — крайние пиксели дублируются.

  7. Тональная компрессия                                                                                                                                                                                               
                                                                                                                                                                                                                        
  Рендер работает в линейном HDR (значения 0...∞). Экран показывает [0...255]. Преобразование:                                                                                                                          
                                                                                                                                                                                                                        
  1. Reinhard-подобная нормировка: img *= 0.5 / mean_luminance — масштабируем так, чтобы средняя яркость ≈ 0.5                                                                                                          
  2. Clip: img = clip(img, 0, 1)                                                                                                                                                                                        
  3. Гамма-коррекция: img = img^(1/2.2) — компенсирует нелинейность монитора (sRGB)                                                                                                                                     
                                                                                                                                                                                                                        
  Ключевой момент: фильтр работает ДО гамма-коррекции в линейном пространстве, что обеспечивает физическую корректность весов и результатов.

  9. Возможные вопросы на защите                                                                                                                                                                                        
                                                                                                                                                                                                                        
  Q: Почему фильтруем только indirect, а не всё изображение?                                                                                                                                                            
  Direct light вычисляется через NEE — он менее шумный. Фильтровать его не нужно, и это сохранило бы детали прямого освещения (резкие тени, блики). Indirect — основная источника шума (случайные bouncing-и).          
                                                                                                                                                                                                                        
  Q: Почему G_r_obj — бинарная маска, а не гауссиан?                                                                                                                                                                    
  Это гарантирует, что цвета разных объектов никогда не смешиваются. Гауссиан дал бы ненулевой вес даже далёким объектам — на границах появился бы ореол. Жёсткая маска — самый надёжный способ, когда у нас есть точный
   object_index.                                                                                                                                                                                                        
                                                                                                                                                                                                                        
  Q: Что если у двух объектов одинаковый материал (пол и потолок оба white)?                                                                                                                                            
  Они получат один object_index, но G_r_normal не даст их смешать: у пола нормаль (0,1,0), у потолка (0,-1,0) — dot = -1, и вес ≈ exp(-2/(2×0.0064)) ≈ 0. Аналогично depth: пол и потолок на разной глубине.            
                                                                                                                                                                                                                        
  Q: Зачем русская рулетка?                                                                                                                                                                                             
  Без неё путь фиксированной длины: вклад дальних bouncing-ов экспоненциально мал, но мы всё равно их трассируем. Русская рулетка случайным образом обрывает путь с вероятностью 1 - p_continue, а throughput           
  компенсирует это делением на p_continue. Матожидание сохраняется → оценка несмещённая.
