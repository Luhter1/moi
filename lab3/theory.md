## Метод трассировки путей (Path Tracing)

---

## 1. Основы метода трассировки путей

### 1.1. Что такое трассировка путей?

**Трассировка путей (Path Tracing)** — это алгоритм глобального освещения, основанный на методе Монте-Карло для численного решения уравнения рендеринга (Rendering Equation). Алгоритм был предложен Джеймсом Каджия (James Kajiya) в 1986 году.

**Основная идея:** Для каждого пикселя изображения выпускается множество лучей из камеры в сцену. Каждый луч, попадая на поверхность, может:
- Поглотиться (завершение пути)
- Отразиться диффузно (рассеянное отражение)
- Отразиться зеркально (направленное отражение)
- Попасть на источник света (добавление вклада освещения)

### 1.2. Уравнение рендеринга (Rendering Equation)

$$L_o(\mathbf{x}, \omega_o) = L_e(\mathbf{x}, \omega_o) + \int_{\Omega} f_r(\mathbf{x}, \omega_i, \omega_o) L_i(\mathbf{x}, \omega_i) (\mathbf{n} \cdot \omega_i) d\omega_i$$

Где:
- $L_o$ — выходящая радиантность (яркость) в точке $\mathbf{x}$ в направлении $\omega_o$
- $L_e$ — собственная эмиссия поверхности (источники света)
- $f_r$ — BRDF (Bidirectional Reflectance Distribution Function)
- $L_i$ — входящая радиантность из направления $\omega_i$
- $\mathbf{n} \cdot \omega_i$ — косинус угла между нормалью и направлением света (закон Ламберта)
- $\Omega$ — полусфера над поверхностью

### 1.3. Метод Монте-Карло в рендеринге

Интеграл в уравнении рендеринга вычисляется методом Монте-Карло:

$$\int f(x) dx \approx \frac{1}{N} \sum_{i=1}^{N} \frac{f(x_i)}{p(x_i)}$$

Где:
- $N$ — количество выборок (лучей на пиксель, SPP)
- $x_i$ — случайные точки выборки
- $p(x_i)$ — функция плотности вероятности (PDF) выборки

## 2. Геометрия сцены и пересечения лучей

### 2.1. Представление геометрии

Сцена представляется в виде **треугольной сетки (mesh)**. Каждый треугольник задаётся тремя вершинами:
- $\mathbf{v_0}, \mathbf{v_1}, \mathbf{v_2}$ — координаты вершин
- $\mathbf{e_1} = \mathbf{v_1} - \mathbf{v_0}$, $\mathbf{e_2} = \mathbf{v_2} - \mathbf{v_0}$ — рёбра треугольника
- $\mathbf{n} = \frac{\mathbf{e_1} \times \mathbf{e_2}}{||\mathbf{e_1} \times \mathbf{e_2}||}$ — нормаль треугольника
- $Area = \frac{1}{2} ||\mathbf{e_1} \times \mathbf{e_2}||$ — площадь треугольника

### 2.2. Алгоритм Мёллера-Трумбора (Möller-Trumbore)

Быстрый алгоритм проверки пересечения луча с треугольником без предварительного вычисления нормали.

**Входные данные:**
- Луч: $\mathbf{r}(t) = \mathbf{o} + t\mathbf{d}$, где $\mathbf{o}$ — начало, $\mathbf{d}$ — направление
- Треугольник: вершины $\mathbf{v_0}, \mathbf{v_1}, \mathbf{v_2}$

**Алгоритм:**
```
h = d × e2
a = e1 · h
if |a| < ε: луч параллелен треугольнику (нет пересечения)

f = 1/a
s = o - v0
u = f · (s · h)
if u < 0 или u > 1: точка вне треугольника

q = s × e1
v = f · (d · q)
if v < 0 или u + v > 1: точка вне треугольника

t = f · (e2 · q)
if t > ε: есть пересечение на расстоянии t
```

**Барицентрические координаты:**
- $u, v$ — барицентрические координаты точки пересечения
- Точка пересечения: $\mathbf{p} = (1-u-v)\mathbf{v_0} + u\mathbf{v_1} + v\mathbf{v_2}$

### 2.3. Ускорение пересечений

В реализации используется **векторизация на NumPy**:
- Все треугольники хранятся в плоских массивах (flat arrays)
- Пересечения вычисляются одновременно для всех треугольников
- Выбирается ближайшее пересечение с минимальным $t$

## 3. Оптические свойства материалов

### 3.1. Модель материала

В реализации используется упрощённая модель с двумя компонентами:

```python
Material:
    diffuse:  np.ndarray  # Коэффициент диффузного отражения (kd)
    specular: np.ndarray  # Коэффициент зеркального отражения (ks)
    emission: np.ndarray  # Эмиссия (для источников света)
```

### 3.2. Закон сохранения энергии

**Физическая корректность:** Суммарное отражение не должно превышать 1:

$$k_d + k_s \leq 1 \quad \text{(покомпонентно для R, G, B)}$$

В коде это обеспечивается в `__post_init__`:
```python
total = self.diffuse + self.specular
if total > 1.0:
    scale = 1.0 / total
    self.diffuse *= scale
    self.specular *= scale
```

### 3.3. Диффузное отражение (Ламберт)

**Закон Ламберта:** Интенсивность отражённого света пропорциональна косинусу угла между направлением света и нормалью.

**BRDF Ламберта:**
$$f_r^{diffuse} = \frac{k_d}{\pi}$$

Где $k_d$ — коэффициент диффузного отражения (альбедо).

**Косинусная выборка полусферы:**
Для эффективной выборки направления используется распределение, пропорциональное косинусу:

$$p(\omega) = \frac{\cos\theta}{\pi}$$

**Метод Мальли (Malley's method):**
```python
u1, u2 = random(), random()
r = sqrt(u1)
theta = 2π * u2
x = r * cos(theta)
y = r * sin(theta)
z = sqrt(1 - u1)  # = cos(θ)
```

### 3.4. Зеркальное отражение

**Закон отражения:** Угол падения равен углу отражения.

$$\mathbf{r} = \mathbf{d} - 2(\mathbf{d} \cdot \mathbf{n})\mathbf{n}$$

Где:
- $\mathbf{d}$ — направление падающего луча
- $\mathbf{n}$ — нормаль поверхности
- $\mathbf{r}$ — направление отражённого луча

В коде:
```python
def reflect(d, n):
    return d - 2.0 * np.dot(d, n) * n
```

## 4. Выборка по значимости и русская рулетка

### 4.1. Выборка по значимости (Importance Sampling)

**Принцип:** Направления выборки выбираются пропорционально их вкладу в результат, что уменьшает дисперсию.

**Для диффузных поверхностей:** Выборка по косинусному распределению (Ламберт).

**Для источников света:** Выборка источника пропорционально его мощности:
```python
powers = [emission * area for each light]
probabilities = powers / sum(powers)
light = random_choice(lights, p=probabilities)
```

### 4.2. Русская рулетка (Russian Roulette)

**Проблема:** При большой глубине трассировки вклад пути становится мал, но вычисления продолжаются.

**Решение:** С некоторой вероятностью прерывать путь, компенсируя это увеличением вклада.

**Алгоритм:**
```python
p_continue = min(max(kd + ks), 0.95)  # Вероятность продолжения
if depth >= rr_start_depth:
    if random() > p_continue:
        break  # Прерываем путь
    throughput /= p_continue  # Компенсируем вклад
```

**Преимущества:**
- Сохраняется математическое ожидание (несмещённая оценка)
- Ускоряется рендеринг
- Автоматически адаптируется к яркости пути

### 4.3. Выбор типа отражения (ROD — Russian Roulette Decision)

```python
p_diff = kd / (kd + ks)  # Вероятность диффузного отражения
if random() < p_diff:
    # Диффузное отражение
else:
    # Зеркальное отражение
```

## 5. Прямое освещение и тени

### 5.1. Next Event Estimation (NEE)

**Проблема:** При чистой трассировке путей сходимость медленная, особенно для прямого освещения.

**Решение:** На каждом шаге явно выбирать точку на источнике света и проверять видимость.

**Формула вклада:**
$$L_{direct} = L_e \cdot \frac{f_r \cdot G}{p_{light}}$$

Где:
- $L_e$ — эмиссия источника
- $f_r = \frac{k_d}{\pi}$ — BRDF Ламберта
- $G = \frac{\cos\theta_s \cos\theta_l}{r^2}$ — геометрический терм
- $p_{light}$ — PDF выборки источника

### 5.2. Геометрический терм (Geometry Term)

$$G(\mathbf{x} \leftrightarrow \mathbf{y}) = \frac{\cos\theta_s \cdot \cos\theta_l}{||\mathbf{x} - \mathbf{y}||^2}$$

Где:
- $\theta_s$ — угол между нормалью поверхности и направлением к свету
- $\theta_l$ — угол между нормалью источника и направлением к поверхности
- $r$ — расстояние между точками

В коде:
```python
cos_surf = dot(normal, to_light_n)      # cos(θs)
cos_light = dot(-light.normal, to_light_n)  # cos(θl)
geom = cos_surf * cos_light / (dist * dist)
```

## 6. Камера и формирование лучей

### 6.1. Модель камеры

**Пинхол-камера (точечная):** Все лучи исходят из одной точки (центра проекции).

**Параметры:**
- `position` — положение камеры
- `look_at` — точка, куда смотрит камера
- `up` — вектор "верха"
- `fov_deg` — поле зрения в градусах
- `width`, `height` — разрешение изображения

### 6.2. Построение системы координат камеры

```python
fwd = normalize(look_at - position)     # Ось Z (направление)
rgt = normalize(cross(fwd, up))         # Ось X (вправо)
u = cross(rgt, fwd)                      # Ось Y (вверх)
```

### 6.3. Формирование лучей

**Нижний левый угол экрана:**
$$\mathbf{lower\_left} = \mathbf{fwd} - \frac{w}{2}\mathbf{rgt} - \frac{h}{2}\mathbf{up}$$

**Направление луча для пикселя (sx, sy):**
$$\mathbf{direction} = \mathbf{lower\_left} + sx \cdot \mathbf{horiz} + sy \cdot \mathbf{vert}$$

### 6.4. Антиалиасинг

**Проблема:** Дискретизация пикселей вызывает ступенчатые края (алиасинг).

**Решение:** Случайное смещение точки выборки внутри пикселя:
```python
sx = (px + random()) / width
sy = (py + random()) / height
```

Это реализует **суперсэмплинг со случайной выборкой**.

## 7. Тональная компрессия и гамма-коррекция

### 7.1. HDR и тональная компрессия

**Проблема:** Реальные яркости сцены могут превышать диапазон [0, 1].

**Решение:** Тональная компрессия (tone mapping).

**Метод в реализации:**
1. Нормировка по средней яркости:
   $$L_{mean} = \text{mean}(L_{nonzero})$$
   $$img = img \cdot \frac{0.5}{L_{mean}}$$

2. Клиппинг:
   $$img = \text{clip}(img, 0, 1)$$

### 7.2. Воспринимаемая яркость (Luminance)

Формула Rec. 709 для перевода RGB в яркость:
$$L = 0.2126 \cdot R + 0.7152 \cdot G + 0.0722 \cdot B$$

### 7.3. Гамма-коррекция

**Проблема:** Мониторы имеют нелинейную характеристику отображения.

**Решение:** Гамма-коррекция с $\gamma = 2.2$:
$$V_{corr} = V^{1/\gamma}$$

В коде:
```python
img_gamma = np.power(img, 1.0 / gamma)
img_uint8 = (img_gamma * 255).astype(np.uint8)
```

## 9. Параллелизация вычислений

### 9.1. Многопроцессорный рендеринг

**Принцип:** Каждая строка изображения независима, можно рендерить параллельно.

**Реализация:**
```python
with mp.Pool(ncpus, initializer=_worker_init, initargs=(tracer, camera)) as pool:
    for py, row in pool.imap_unordered(_render_row, [(py, spp) for py in range(H)]):
        hdr_buffer[py] = row
```

### 9.2. Инициализация воркеров

Каждый процесс получает свою копию трассировщика и камеры:
```python
def _worker_init(tracer, camera):
    global _w_tracer, _w_camera
    _w_tracer = tracer
    _w_camera = camera
    np.random.seed(os.getpid())  # Уникальный seed
```

## 10. Корнельская коробка (Cornell Box)

### 10.1. Что это?

Стандартная тестовая сцена для проверки алгоритмов глобального освещения:
- Комната с белыми стенами
- Красная и зелёная стены по бокам
- Два блока разной формы
- Источник света на потолке

### 10.2. Параметры в реализации

```python
# Комната: [0,1] × [0,1] × [0,1]
# Стены: 5 квадов (пол, потолок, зад, лево, право)
# Блоки: 
#   - Высокий зеркальный (повёрнут на -30°)
#   - Низкий смешанный (повёрнут на 20°)
# Свет: квад на потолке [0.35, 0.65] × [0.35, 0.65]
```

## 11. Вопросы для самопроверки

### 11.1. Базовые вопросы

1. **Что такое уравнение рендеринга?**
   - Интегральное уравнение, описывающее перенос света в сцене

2. **Почему метод называется "трассировка путей"?**
   - Потому что трассируется путь луча через множественные отражения

3. **В чём разница между трассировкой лучей и трассировкой путей?**
   - Ray tracing: только зеркальные отражения, прямое освещение
   - Path tracing: диффузные + зеркальные, глобальное освещение (Монте-Карло)

### 11.2. Математические вопросы

4. **Зачем делить на PDF в методе Монте-Карло?**
   - Для получения несмещённой оценки интеграла

5. **Как работает русская рулетка?**
   - С вероятностью $1-p$ прерываем путь, с вероятностью $p$ продолжаем и делим вклад на $p$

6. **Почему BRDF Ламберта равна $k_d/\pi$?**
   - Из условия сохранения энергии: $\int f_r \cos\theta d\omega = k_d$

### 11.3. Вопросы реализации

7. **Зачем нужен антиалиасинг?**
   - Устранение ступенчатых краёв из-за дискретизации пикселей

8. **Как работает алгоритм Мёллера-Трумбора?**
   - Использует барицентрические координаты для проверки пересечения

9. **Почему используется векторизация NumPy?**
   - Ускорение вычислений за счёт одновременной обработки всех треугольников

10. **Как обеспечивается сохранение энергии?**
    - Нормализация $k_d + k_s \leq 1$ в конструкторе Material
